调用 git-workflow-and-versioning 技能。引导完成一次规范化的 Git 提交。

1. **检查 Staged 变更** - 确认变更已暂存，审查 diff 内容
2. **原子性检查** - 确认提交只做一件逻辑上完整的事，关注点未混杂
3. **提交前检查** - 无密钥泄露、测试通过、Lint 通过、类型检查通过
4. **生成提交消息** - `<type>: <中文简述>` 格式，解释“为什么”而非“做了什么”
5. **输出变更摘要** - 修改了什么 / 有意未修改什么 / 潜在关注点

类型前缀：`feat`（新功能）| `fix`（Bug 修复）| `refactor`（重构）| `test`（测试）| `docs`（文档）| `chore`（工具链/配置）

## 使用方式

在 `/commit` 后描述你的提交意图，我将引导你完成规范化提交流程。

### 示例

```
# 常规提交
/commit 提交当前的用户注册功能实现

# 审查后提交
/commit 检查 staged 变更，确认没有混入无关改动后提交

# 批量变更拆分提交
/commit 当前 staged 了 5 个文件的改动，帮我拆分为原子提交
```
