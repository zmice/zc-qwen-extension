调用 deprecation-and-migration 技能。渐进迁移，安全移除旧代码。

1. 影响评估 — 识别所有受影响的调用方，评估范围和风险
2. 迁移计划 — 编写迁移指南，设计兼容层/适配器
3. 执行 — 添加废弃标注，实现新 API，逐步迁移调用方
4. 验证 — 确认全部迁移完成，回归测试通过
5. 清理 — 移除废弃代码，设定移除时间表

## 使用方式

在 `/migrate` 后描述你要迁移或废弃的内容，我将制定详细的迁移方案。

### 示例

```
# 框架迁移
/migrate 前端项目从 Vue 2 迁移到 Vue 3，涉及 200+ 组件

# API 废弃
/migrate 废弃旧的 v1 支付接口，引导用户迁移到 v2

# 数据库迁移
/migrate 从 MySQL 迁移到 PostgreSQL，需要零停机方案
```
